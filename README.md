# fabletradebot V4 — 고래 모드: 단일 집중 · 국면 인식 · 이벤트+연속 진입

**설계 철학**: 과거 데이터 최적화가 아니라 **선험적 완전성** — 모든 시장 국면에
대응하는 교과서-표준 진입 세트를 설계하고, 페이퍼 전방 트랙이 각 플레이북을
심판한다. TF 위계는 고래 관점: **4H가 기본 판단 축**(셋업 포착), **1D는 참고**
(국면·편향), **1H는 정밀 타점과 채점 전용**. 진입은 두 종류: 이벤트 확인형
(크로싱 순간)과 **연속 차트-상태형**(크로싱을 놓쳐도 현재 차트가 매집 구조면 진입).

이 시스템은 **고래 모드(V4)**로 운영된다: 포트폴리오 분산을 버리고, 매 봉 전
유니버스 후보 중 **가장 확신도 높은 단 하나**의 코인에 **전체 자산**을 확신도별
레버리지로 몰아넣는다. 진입한 자리는 청산까지 홀드하며(다른 코인 신호가 직접
밀어내지 않음), 매시 재채점되는 실시간 확신도(hold_confidence)가 승자는 모멘텀
페이드로 익절, 패자는 국면/모멘텀이 명확히 돌아섰을 때 구조적 손절보다 앞서
손실을 줄인다.

> **정직한 리스크 선언 (반드시 읽을 것)**: 고정 리스크로 손실을 상수화하는
> 포트폴리오 방식과 달리, 고래 모드의 손절 1회는 **레버리지×손절폭**만큼 계좌를
> 깎는다(예: 10x·손절 2% = −20%). 청산-거리 캡(`L_liq`) 때문에 이 값은 구조적으로
> **최대 ~35%**로 상한이 걸리지만, 단일 포지션 집중이라는 설계 자체가 이 저장소가
> 다뤄본 어떤 방식보다도 **높은 꼬리 수익·높은 파산 위험**을 동시에 가진다. 이
> 숫자를 숨기지 않는 것이 이 시스템의 핵심 가치다(§7).

## 트레이딩 로직 — 구체 설명

### 0. 파이프라인 한눈에

```
매 1H 마감(+3분, cron-job.org → GitHub Actions)
 ├─ OKX 1H 캔들+8h 펀딩 증분 수집 (CSV 캐시, 마감봉만)
 ├─ BTC 1D → 광역 국면 분류 (TREND/RANGE/HIGH_VOL/CRISIS + 히스테리시스) [1D=참고]
 ├─ 코인별 독립 스캔: 마감 4H봉의 크로싱 이벤트로 슬롯 무장 [4H=기본]
 │    → 이후 4시간 내 1H 확인봉이 격발 → 확신도(0~1) → 후보 [1H=정밀 타점]
 ├─ 교차-코인 선택: 후보 중 확신도 최고 하나만 유일한 자리를 차지
 │    (자리가 이미 차 있으면 신호는 대기 — 밀어내지 않음)
 ├─ 리스크 게이트 (국면·DD거버너·서킷) 통과 시 다음 1H 시가에 진입
 │    (풀마진, 확신도 티어 레버리지, 청산안전 캡)
 ├─ 보유 중: 8×ATR 샹들리에 트레일 / 4H bias 플립 청산 / CRISIS 청산
 │    매 봉 재채점(hold_confidence) → 승자 모멘텀 페이드(SignalFade) 익절,
 │    패자 국면/모멘텀 붕괴(LossFade) 손절보다 앞서 청산 [§7·§10]
 └─ Telegram 알림 + Notion 저널 + 상태 커밋 (롤링 앵커 = 새 봉만 증분 처리, §11)
```

### 1. 데이터와 무(無)룩어헤드 규약

- OKX 무기한 선물 1H OHLCV + 8h 펀딩. 4H/1D는 1H에서 리샘플(부분 봉 제외).
- 판단 시각 t에는 **t에 이미 마감된 봉만** 사용. 상위 TF 피처는 "마감 시각 ≤ t"인
  봉만 투영(`closed_asof_1h`). pytest가 데이터 절단 불변성·돌파선 사전성으로 증명.

### 2. 국면 분류 — 자산별 방향 명시형 5국면 (V3)

**왜 자산별인가 (FN 제거)**: BTC 광역 국면 하나로 전 유니버스를 게이트하면 BTC가
횡보일 때 홀로 추세인 알트의 추세 셋업이 통째로 막히는 구조적 **False Negative**.
각 코인을 **자기 일봉으로 독립 분류**하고, 시스템 리스크인 **CRISIS만 BTC에서
전역 상속**한다(위기는 언제나 전역이므로).

각 자산의 1D봉에서 (우선순위 순):

| 상태 | 판정 | 진입 정책 | L 상한 |
|---|---|---|---|
| **CRISIS** | 5D<−12% 또는 1D<−7% 또는 (변동성>90p & 5D<−8%), 또는 BTC가 CRISIS | 신규 금지 + 보유 즉시 청산 | 0 |
| **HIGH_VOL** | 실현변동성(20D) 1년 백분위 > 80 | 이벤트 확인형(OSC)만, 절반 리스크 | 3x |
| **TREND_UP** | \|EMA20−EMA50\|>0.5ATR & 종가>EMA100 & EMA20>EMA50 | 롱 추세 플레이북(BRK_L/PBK_L/RCL_L) | 10x |
| **TREND_DOWN** | 〃 반대(종가<EMA100 & EMA20<EMA50) | 숏 추세 플레이북(BRK_S/PBK_S/RCL_S) | 10x |
| **RANGE** | 그 외 | 회귀 플레이북(OSC/BND) 양방향 | 5x |

방향을 상태에 새겨(**TREND_UP ≠ TREND_DOWN**) 롱/숏 플레이북이 정확한 국면에서만
무장된다 — 상승추세에서 숏 셋업이 뜨는 False Positive를 원천 차단. 전환은 새 상태
2연속 1D봉 확인(CRISIS 진입 즉시, 해제 3봉) — 채터링 방지.

#### 2-1. 국면 → 플레이북 매핑

```
TREND_UP    → BRK_L(돌파) · PBK_L(연속 눌림매집) · RCL_L(EMA20 재탈환)     [롱만]
TREND_DOWN  → BRK_S · PBK_S · RCL_S                                        [숏만]
RANGE       → OSC_L/S(RSI 30/70 재돌파) · BND_L/S(밴드 재진입) · BRK(돌파) [양방향]
HIGH_VOL    → OSC_L/S만 (변동성 국면은 이벤트 확인 없이는 진입 안 함)
CRISIS      → 없음 (관망 + 청산)
```

### 3. 진입 전략 — 완전 커버리지 플레이북 매트릭스

**설계 원리 4개 (전부 선험, 교과서 표준값, 최적화 없음):**

1. **두 종류의 진입** — (a) **이벤트 확인형**: 레벨이 아니라 *크로싱*. 과매도라서
   사는 게 아니라 과매도에서 **빠져나오는 순간**(RSI 30 상향 재돌파) 산다. (b)
   **연속 차트-상태형(PBK)**: 크로싱을 놓쳐도 매 봉 현재 차트를 읽어, 추세 중
   가격이 가치 구간(4H EMA20~EMA50)으로 눌리고 모멘텀이 리셋됐을 때 1H 확인봉에
   진입. **고래가 눌림에서 조용히 매집하듯** 이벤트 없이도 타점을 잡는다.
2. **무장(arm) → 격발(trigger) 2단 구조** — 마감된 **4H 봉**에서 셋업(크로싱 또는
   가치 구간 진입)이 성립하면 슬롯이 무장되고, **1H 확인봉**(방향 실체봉이 직전
   1H 고/저 돌파)이 격발한다. 4H가 판단, 1H가 방아쇠.
3. **국면-방향 정합** — 각 슬롯은 자기 국면·방향에서만 무장(§2-1). TREND_UP에서
   숏 슬롯은 아예 안 뜬다(False Positive 원천 차단).
4. **청산은 스타일 종속** — 추세형(BRK/PBK/RCL)은 트레일8+바이어스 플립, 회귀형
   (OSC/BND)은 고정 TP 1.5R + 시간청산 48h.

| 슬롯 | 국면 | 셋업 (무장 조건) | 트리거 종류 | 스타일 | 스케일 |
|---|---|---|---|---|---|
| **BRK_L/S** | TREND·RANGE | 7일 신고/신저가 돌파 (1D·4H·BTC 정렬) | 이벤트 | 스윙 트레일 | L 1.0 / S 0.2 |
| **PBK_L/S** | TREND | 가치구간(EMA20~EMA50) 눌림 + 1H RSI 40~60 리셋 | **연속 상태** | 스윙 트레일 | 0.2 |
| **RCL_L/S** | TREND | 4H 종가가 EMA20 **재탈환/재상실** (1D 동의) | 이벤트 | 스윙 트레일 | 0.2 |
| **OSC_L/S** | RANGE·HIGH_VOL | RSI(14,4H) **30 상향 / 70 하향 재돌파** | 이벤트 | 데이 TP+시간 | 0.2 |
| **BND_L/S** | RANGE | 4H 종가 2σ 밴드 밖→**안 복귀** | 이벤트 | 데이 TP+시간 | 0.2 |

- 회귀형(OSC/BND)은 확정된 1D 추세와 **역행 금지**.
- 손절: 셋업 4H봉/스윙 극단 ∓ 0.5×ATR4H(추세형은 8봉 스윙 ∓0.6×ATR1H), 전역
  2×ATR1H 하한.
- CRISIS: 전 슬롯 무장 해제 + 보유 청산.
- 리스크 스케일(위 표 마지막 열)은 **슬롯이 신호를 낼 확신도의 사전 가중치**일
  뿐 — 고래 모드의 실제 포지션 사이즈는 §5의 풀마진 사이징이 전적으로 결정한다
  (스케일은 여러 후보가 동시에 뜰 때의 교차-코인 확신도 비교에 반영되지 않는다).
- **Upgrade 지배 규칙**: 검증 슬롯(BRK_L) 신호는 같은 코인의 실험 포지션을
  청산·승격한다(역방향 선점 불가) — 실험이 코어를 잠식 못 하게 하는 기계적 방화벽.

#### 3-0. FN/FP 관점의 필터 설계 (E14 — 신호 정밀도)

진입 신호는 "다음 24h가 비용 허들(0.4%) 이상 유리하게 움직이나?"의 이진 분류기다.
각 필터가 **False Positive**(진입했는데 틀림)와 **False Negative**(막았는데
승자였음)를 어떻게 맞바꾸는지 `scripts/fn_fp.py`로 측정한다. BTC 7일 돌파(raw)에
필터를 하나씩 얹은 결과:

| 필터 | 통과분 승률(=1−FP) | 거부분 승률(FN 비용) | 판정 |
|---|---|---|---|
| raw 돌파(무필터) | 0.55 | — | 기준 |
| + 1D 추세 정렬 | 0.56 | 0.54 | 값 있음 |
| + 4H 추세 정렬 | **0.57** | **0.37** | 강함 (거부분이 훨씬 못 이김) |
| + 거래량 ≥1.2× | 0.56 | 0.33 | 강함 |

각 필터가 FP를 낮추면서 거부한 진입(FN)은 실제로 덜 이긴다 → 필터가 값을 한다.
반대로 숏 슬롯(PBK_S FP 0.56, OSC_S FP 0.63)은 FP가 높아 실험 스케일 유지 근거.

#### 3-1. BRK_L 상세 (백테스트 검증이 있는 유일한 슬롯)

다음 **전부** 충족 시 후보 (1H 봉 t 마감 기준):

```
① close(t) > max(high[t-168 .. t-1])          # 직전 7일 신고가 돌파
② 1D bias = up   (EMA20>EMA50 & close>EMA100) # 자산 자체 일봉 추세
③ 4H bias = up   (EMA20>EMA50)                # 중간 TF 정렬
④ BTC 1D 방향 = up                            # 광역 순풍
⑤ volume(t) ≥ 1.2 × median(volume, 48)        # 참여 확인
⑥ 국면 ∈ {TREND, RANGE}
```

**롱 전용인 이유(측정)**: 최근 16개월, 신저가 돌파 숏은 12/12 자산에서 진입 방향과
반대로 +3.1%/72h(숏스퀴즈 지배). 약세장은 관망이 포지션이다.

확신도: `conf = 0.55×base + 0.25×국면적합 + 0.20×TF정렬 + 펀딩수정자`
(base = 돌파마진·사전압축(BB폭 백분위)·돌파봉 실체·거래량의 평균,
펀딩수정자 = 숏 크라우딩(z<−1.5) +0.05 / 롱 크라우딩(z>+1.5) −0.10).
`conf ≥ 0.55`만 진입 자격. **주의: conf는 진입 자격 필터 + 코인 선택 랭킹(§4)에
쓰일 뿐, 포지션 사이즈는 결정하지 않는다** — 설계 측정에서 conf가 R을 랭킹하지
못했음(corr 0.005)을 그대로 반영한 결정. 고래 모드에서는 conf가 대신 **레버리지
티어**를 정한다(§5) — 사이징이 아니라 사이징의 입력인 레버리지 캡이라는 점에서
여전히 "conf가 R을 예측하지 않는다"는 원칙과 모순되지 않는다.

### 4. 손절 배치 (E5 측정 반영)

```
SL = min( 돌파선 − 0.6×ATR1H,  close − 2.0×ATR1H )   # 더 먼 쪽
```
1H 노이즈 밴드(≈2×ATR) 안의 손절은 스윕당한다는 측정 결과로 변동성 하한 강제.
진입 후 손절은 타이트닝만 허용(절대 안 넓힘).

### 5. 코인 선택과 확신도 → 레버리지 · 풀마진 사이징

**단 하나의 자리.** 같은 봉에 여러 코인 후보가 뜨면 **확신도 내림차순**으로
정렬해 최고 확신도 후보가 먼저 자리를 가져간다(교차-코인 선택). 이미 자리가
차 있으면 검증 슬롯(BRK_L)만 실험 포지션을 밀어낼 수 있고(Upgrade, §3), 그 외엔
대기한다 — 진입한 자리는 청산까지 홀드된다.

확신도는 목표 레버리지 티어로 사상된다:

```
conf ≥ 0.80 → 10x      conf ≥ 0.70 → 5x
conf ≥ 0.62 →  3x      conf ≥ 0.55 → 2x   (0.55 미만은 진입 자격 없음)
```

포트폴리오 방식이 손절폭에서 역산해 고정 리스크를 정하는 것과 달리, 고래 모드는
**전체 자산을 증거금**으로 넣는다:

```
확신도 티어 레버리지 L_conf  (위 표)
최종 레버리지 L = floor_tier( min( L_conf, 국면 캡, 자산 캡, L_liq ) )
   L_liq = 1 / (3×stop_frac + 1.5%)          # 청산 거리 ≥ 3×손절 거리 (불변식)
   floor_tier: 2/3/5/10 표준 계단으로 내림

명목(notional) = equity × L                  # 전체 자산 × 레버리지
증거금(margin) = equity                       # = 명목 / L  (계좌 전부가 증거금)
손절 시 손실   = 명목 × stop_frac = equity × L × stop_frac
```

**핵심 — 청산 안전 불변식은 절대적이다.** L이 `L_liq`로 캡되므로 **손절폭이
넓으면 레버리지가 자동으로 낮은 티어로 내려간다**: 예컨대 확신도가 10x를 원해도
손절폭이 8%면 `L_liq ≈ 3.9` → **3x로 강등**돼 청산가가 손절보다 항상 3배 멀리
앉는다. 엔진은 `size_position`에서 매 진입 시 "손절이 청산보다 먼저"를 assert한다.

**정직한 리스크 경고.** 손절 한 번이 `L×stop_frac`만큼 계좌를 깎는다 — 10x·손절
2%면 **−20%**, 5x·3%면 −15%. `L_liq` 캡 때문에 `L×stop_frac ≤ 1/3`이 구조적
상한이라 단일 손절 최대 손실은 계좌의 ~35% 부근이다. 이 프로파일은 저장소가
다뤄본 방식 중 **최고 파산 위험**이고 페이퍼 전용이다(§11).

### 6. 리스크 가드 (분산 없이 단일 포지션을 지키는 장치)

동시 포지션 상한·상관 경보는 자리가 하나뿐이라 의미가 없어 열어두는 대신, 아래
장치가 계좌를 지킨다:

- **DD 거버너**: 드로다운 −10%부터 **증거금**을 절반으로(`margin_frac`, 레버리지·
  청산 거리는 불변) — 상관 경보 시에도 동일 반감. −20%부터 신규 진입 정지,
  −15%에서 해제.
- **24h 서킷 브레이커**: 24h 실현손실 ≥ 4% equity → 24h 신규 정지.
- **CRISIS 국면**: 신규 금지 + 보유 즉시 청산.
- **쿨다운**: 청산 후 해당 코인 12봉 재진입 금지(RSI<20 지속 구간의 칼날 재포착
  방지).

> **완결성 수정(감사 로그)**: 초기 구현에서 `dd_half`(드로다운 시 사이즈 반감)와
> 상관경보 반감이 **풀마진 사이징에서 무시**되고 있었다(`risk_frac`을 거치는 두
> 컨트롤이 full-margin 경로에서 미적용). 실제 캐시 백테스트에서 진입 8건 중 3건이
> 이 반감을 발동시켰으나 전액 사이징돼 있었다. 수정: 두 컨트롤이 이제 **증거금
> 비율**(`margin_frac`)로 반영돼 드로다운·상관 국면에 노출을 줄인다(레버리지·
> 청산 거리 불변). 결과 MDD −35%→−27%.

### 7. 청산 (측정으로 선택 — 부분익절·시간손절은 승자를 잘라 기각)

1. **초기 SL** (§4)
2. **8×ATR1H 샹들리에 트레일**: 진입 직후부터, `max(최고 종가 − 8×ATR, 현재 SL)`
3. **4H bias 플립**: EMA20<EMA50 2연속 4H봉 → 종가 청산 (추세 종료 판정)
4. **CRISIS 국면** → 즉시 청산
5. **SignalFade**: 매시 재채점에서 **승자**가 모멘텀을 잃거나 확신도가 붕괴하면
   익절 — 상세 §10. `hold_conf_exit=0.50`.
6. **LossFade**: 매시 재채점에서 **패자**의 확신도가 붕괴 상태로 2봉 연속 머물면
   구조적 손절보다 앞서 손실을 줄인다 — 상세 §10. `hold_loss_exit=0.50`.
- 같은 봉에서 SL·상방이 겹치면 **항상 SL 먼저**(보수적 체결 가정).
- 청산 우선순위: CRISIS > SignalFade > LossFade > bias플립 > (시간손절, 이 슬롯
  매트릭스에서는 대부분 비활성).

### 8. 비용 모델

테이커 0.05%/side + 슬리피지 0.02~0.08%/side(자산군별, 체결가에 반영) + 실측 펀딩
(히스토리 없는 과거 구간은 8h당 0.01% 보수 드래그). 비용 2배 스트레스에서도
설계 구간 총수익 유지가 검증 게이트(G6)다.

### 9. 채점(scoring) 워크플로우 — 두 층위

"채점"은 두 가지를 뜻하며 둘 다 매 실행마다 돈다.

**층위 A — 시간별 포지션 재채점 (관리로 직결).** 매시 리플레이가 끝나면 오픈
포지션을 최신 봉으로 마크투마켓(미실현 R·가격손익%·레버리지손익%·보유시간)하고,
**추가로 실시간 확신도(hold_confidence)를 재계산**한다. 이는 "지금 이 포지션의
근거가 아직 살아있나?"의 연속 점수 [0,1], **방향 대칭**이라 승자·패자 모두에게
같은 잣대를 적용한다:

```
hold_confidence(d) = 0.45×MTF정렬 + 0.30×국면적합 + 0.25×4H모멘텀
  MTF정렬  : 1D·4H·BTC가 포지션 방향과 몇 개나 일치하나 (0, ⅓, ⅔, 1)
  국면적합 : 추세(방향일치)=1.0, RANGE=0.5, HIGH_VOL=0.35, 반대추세/CRISIS=0
  4H모멘텀 : 종가가 4H EMA20에서 우호 방향으로 얼마나(ATR 단위 연속값) +
             RSI(4H)가 우호 방향
```

이 값은 시간별 채점 리포트(`open_report`, 예: `... hold_conf 0.66`)와 Notion 오픈
행의 Note에 매시 갱신돼 **"현재 확신도 평가"가 눈에 보인다**. 엔진(백테스트=라이브
동일 코드)은 이 재채점으로 **두 가지** 청산을 판단한다:

```
SignalFade (승자만, unreal_r > 0)  — hold_conf_min_r(=1R) 이상 달린 뒤
  ① 모멘텀 상실 : 최고 도달 R에서 hold_giveback(=50%)를 반납  (고점 러너가 꺾임)
  ② 새로운 상황 : hold_confidence < hold_conf_exit(=0.50)가 2봉 연속 (레짐/정렬 반전)

LossFade (패자만, unreal_r < 0) — 구조적 SL이 담당하던 영역의 보완
  국면/모멘텀 명확 붕괴 : hold_confidence < hold_loss_exit(=0.50)가 2봉 연속
  → 풀 손절(최대 ~35%, §5)까지 끌려가지 않고 더 작은 손실로 청산
```

지연 신호인 4H bias 플립보다 **먼저** 작동해 되돌림/추가 하락 전에 결과를 확정하고,
청산 후 자리가 비면 §5의 선택 로직이 다음 최고 확신도 코인으로 넘어간다.

**설계 근거(측정, G5 민감도 포함)**: SignalFade는 캐시 백테스트에서 2회 발동, 평균
+0.80R·승률 100% — 승자를 확정하는 방향으로 작동함을 확인(표본 작아 전방 트랙이
최종 심판). LossFade는 설계 구간에서 `hold_loss_exit` ∈ {0.30, 0.35, 0.40, 0.45,
0.50, 0.55, 0.60}로 민감도를 돌린 결과, **0.40 미만은 바인딩하지 않고(비활성),
0.45~0.60에서는 -0.2~-0.35R로 손실을 조기 확정하면서도 기대값 부호가 전 구간에서
유지**됨을 확인 — 엣지를 파괴하지 않는 무해한 컷으로 판단해 0.50을 채택. 단, 고래
모드 자체에서는 표본이 너무 작아(0회 발동, 설계 구간) 이 메커니즘의 최종 검증은
페이퍼 전방 트랙(층위 B)에 있다.

**층위 B — 전방 트랙 채점 (슬롯·프로파일 심사).** `score_report`가 청산된
트레이드를 setup별·국면별·확신도 티어별·코인별·**청산사유별**로 분해해 기대값(R)을
측정한다. SignalFade/LossFade는 각각 자체 사유 버킷으로 잡혀 "이 청산이 실제로
값을 하나"가 전방 성적으로 심판된다(§과적합 규율).

고래 모드는 **단일-포지션·풀마진** 구조라 포트폴리오 방식의 1000경로 몬테카를로

구간 없음 — 7거래·MDD −27%·최종자산 상승, §6 완결성 수정 반영 후). **그럼에도
고래 모드는 저장소가 다뤄본 방식 중 최고 꼬리·최고 파산 위험**(단일 손절이 계좌의

페이퍼 전방 트랙(§9 층위 B)이 setup·청산사유별로 기대값을 쌓아 내린다.

### 과적합 방지 규율 (이 리포의 헌법)

- 모든 탐색은 설계 구간에서만. 홀드아웃 재실행 시 문서 공개.
- 실패 실험 전부 기록(EXPERIMENTS.md E1~E14): 신호 4계열·숏 브레이크다운·확신도
  사이징·페이드/레인지 — 전부 측정 기각.
- 새 메커니즘은 "사전 표준값 + 사후 ±20% 섭동 확인"만 허용, 격자 최적화 금지.
- 성패 판정은 백테스트가 아니라 **페이퍼 전방 트랙**.
