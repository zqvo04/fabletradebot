# fabletradebot V3 — 자산별 국면 인식 · 이벤트+연속 진입 · 레버리지 프로파일

**설계 철학**: 과거 데이터 최적화가 아니라 **선험적 완전성** — 모든 시장 국면에
대응하는 교과서-표준 진입 세트를 설계하고, 페이퍼 전방 트랙이 각 플레이북을
심판한다. TF 위계는 고래 관점: **4H가 기본 판단 축**(셋업 포착), **1D는 참고**
(국면·편향), **1H는 정밀 타점과 채점 전용**. 진입은 두 종류: 이벤트 확인형
(크로싱 순간)과 **연속 차트-상태형**(크로싱을 놓쳐도 현재 차트가 매집 구조면 진입).

> **레버리지 목표에 대한 정직한 선언 (반드시 읽을 것)**: 목표는 레버리지를 적극
> 활용한 월 100%다. 진실은 — **월 100%를 "기대값"으로 만들면서 파산 확률을 0
> 근처로 두는 파라미터는 수학적으로 존재하지 않는다**(월샤프 »3 필요, 본 시스템은
> ~1). 존재하는 것은 생존↔꼬리수익을 맞바꾸는 **프로파일 스펙트럼**이고, 각각의
> 파산 확률을 몬테카를로로 명시해 눈 뜨고 고르게 한다(§10). `base`는 생존 게이트
> 통과(최고의 달 +41%), `turbo/max`는 +100%대 달이 나오지만 P(파산성 −50% DD)가
> 58~86%다. 이 숫자를 숨기지 않는 것이 이 시스템의 핵심 가치다.

고확신 셋업을 1H 마감마다 코인별 독립 스캔해 discrete 진입하고, 트레이드가
**스스로 작동을 증명하면**(BTC 한정) 피라미딩+리스크 부스트로 공격적으로 키우며,
손절이 청산보다 항상 먼저 맞도록 파산을 구조적으로 봉쇄하는 시스템.

## 트레이딩 로직 — 구체 설명

### 0. 파이프라인 한눈에

```
매 1H 마감(+3분, cron-job.org → GitHub Actions)
 ├─ OKX 1H 캔들+8h 펀딩 증분 수집 (CSV 캐시, 마감봉만)
 ├─ BTC 1D → 광역 국면 분류 (TREND/RANGE/HIGH_VOL/CRISIS + 히스테리시스) [1D=참고]
 ├─ 코인별 독립 스캔: 마감 4H봉의 크로싱 이벤트로 슬롯 무장 [4H=기본]
 │    → 이후 4시간 내 1H 확인봉이 격발 → 확신도(0~1) → 후보 [1H=정밀 타점]
 ├─ 리스크 게이트 (국면·상관·DD거버너·서킷·동시노출) 통과 시
 │    다음 1H 시가에 진입 (리스크 0.55%/유닛, 청산안전 레버리지)
 ├─ 보유 중: 8×ATR 샹들리에 트레일 / 4H bias 플립 청산 / CRISIS 청산
 │    (BTC만) +2R·+4R 도달 시 피라미딩 유닛 추가
 └─ Telegram 알림 + Notion 저널 + 상태 커밋 (결정론적 리플레이 = 상태는 데이터뿐)
```

### 1. 데이터와 무(無)룩어헤드 규약

- OKX 무기한 선물 1H OHLCV + 8h 펀딩. 4H/1D는 1H에서 리샘플(부분 봉 제외).
- 판단 시각 t에는 **t에 이미 마감된 봉만** 사용. 상위 TF 피처는 "마감 시각 ≤ t"인
  봉만 투영(`closed_asof_1h`). pytest가 데이터 절단 불변성·돌파선 사전성으로 증명.

### 2. 국면 분류 — 자산별 방향 명시형 5국면 (V3)

**왜 자산별인가 (FN 제거)**: V2까지는 BTC 광역 국면 하나로 13개 코인을 몽땅
게이트했다 — BTC가 횡보면 홀로 추세인 알트의 추세 셋업이 통째로 막히는 구조적
**False Negative**. V3는 각 코인을 **자기 일봉으로 독립 분류**하고, 시스템
리스크인 **CRISIS만 BTC에서 전역 상속**한다(위기는 언제나 전역이므로).

각 자산의 1D봉에서 (우선순위 순):

| 상태 | 판정 | 진입 정책 | L 상한 |
|---|---|---|---|
| **CRISIS** | 5D<−12% 또는 1D<−7% 또는 (변동성>90p & 5D<−8%), 또는 BTC가 CRISIS | 신규 금지 + 보유 즉시 청산 | 0 |
| **HIGH_VOL** | 실현변동성(20D) 1년 백분위 > 80 | 이벤트 확인형(OSC)만, 절반 리스크 | 3x |
| **TREND_UP** | \|EMA20−EMA50\|>0.5ATR & 종가>EMA100 & EMA20>EMA50 | 롱 추세 플레이북(BRK_L/PBK_L/RCL_L) | 10x |
| **TREND_DOWN** | 〃 반대(종가<EMA100 & EMA20<EMA50) | 숏 추세 플레이북(BRK_S/PBK_S/RCL_S) | 10x |
| **RANGE** | 그 외 | 회귀 플레이북(OSC/BND) 양방향 | 5x |

방향을 상태에 새겨(**TREND_UP ≠ TREND_DOWN**) 롱/숏 플레이북이 정확한 국면에서만
무장된다 — 상승추세에서 숏 셋업이 뜨는 False Positive를 원천 차단. 전환은 새 상태
2연속 1D봉 확인(CRISIS 진입 즉시, 해제 3봉) — 채터링 방지.

### 2-1. 국면 → 플레이북 매핑 (어느 국면에 무엇을 켜는가)

```
TREND_UP    → BRK_L(돌파) · PBK_L(연속 눌림매집) · RCL_L(EMA20 재탈환)     [롱만]
TREND_DOWN  → BRK_S · PBK_S · RCL_S                                        [숏만]
RANGE       → OSC_L/S(RSI 30/70 재돌파) · BND_L/S(밴드 재진입) · BRK(돌파) [양방향]
HIGH_VOL    → OSC_L/S만 (변동성 국면은 이벤트 확인 없이는 진입 안 함)
CRISIS      → 없음 (관망 + 청산)
```

### 3. 진입 전략 — V3 완전 커버리지 플레이북 매트릭스

**설계 원리 4개 (전부 선험, 교과서 표준값, 최적화 없음):**

1. **두 종류의 진입** — (a) **이벤트 확인형**: 레벨이 아니라 *크로싱*. 과매도라서
   사는 게 아니라 과매도에서 **빠져나오는 순간**(RSI 30 상향 재돌파) 산다. (b)
   **연속 차트-상태형(PBK)**: 크로싱을 놓쳐도 매 봉 현재 차트를 읽어, 추세 중
   가격이 가치 구간(4H EMA20~EMA50)으로 눌리고 모멘텀이 리셋됐을 때 1H 확인봉에
   진입. **고래가 눌림에서 조용히 매집하듯** 이벤트 없이도 타점을 잡는다.
2. **무장(arm) → 격발(trigger) 2단 구조** — 마감된 **4H 봉**에서 셋업(크로싱 또는
   가치 구간 진입)이 성립하면 슬롯이 무장되고, **1H 확인봉**(방향 실체봉이 직전
   1H 고/저 돌파)이 격발한다. 4H가 판단, 1H가 방아쇠.
3. **국면-방향 정합** — 각 슬롯은 자기 국면·방향에서만 무장(§2-1). TREND_UP에서
   숏 슬롯은 아예 안 뜬다(False Positive 원천 차단).
4. **청산은 스타일 종속** — 추세형(BRK/PBK/RCL)은 트레일8+바이어스 플립, 회귀형
   (OSC/BND)은 고정 TP 1.5R + 시간청산 48h.

| 슬롯 | 국면 | 셋업 (무장 조건) | 트리거 종류 | 스타일 | 스케일 |
|---|---|---|---|---|---|
| **BRK_L/S** | TREND·RANGE | 7일 신고/신저가 돌파 (1D·4H·BTC 정렬) | 이벤트 | 스윙 트레일 | L 1.0 / S 0.2 |
| **PBK_L/S** | TREND | 가치구간(EMA20~EMA50) 눌림 + 1H RSI 40~60 리셋 | **연속 상태** | 스윙 트레일 | 0.2 |
| **RCL_L/S** | TREND | 4H 종가가 EMA20 **재탈환/재상실** (1D 동의) | 이벤트 | 스윙 트레일 | 0.2 |
| **OSC_L/S** | RANGE·HIGH_VOL | RSI(14,4H) **30 상향 / 70 하향 재돌파** | 이벤트 | 데이 TP+시간 | 0.2 |
| **BND_L/S** | RANGE | 4H 종가 2σ 밴드 밖→**안 복귀** | 이벤트 | 데이 TP+시간 | 0.2 |

- 회귀형(OSC/BND)은 확정된 1D 추세와 **역행 금지**.
- 손절: 셋업 4H봉/스윙 극단 ∓ 0.5×ATR4H(추세형은 8봉 스윙 ∓0.6×ATR1H), 전역
  2×ATR1H 하한.
- CRISIS: 전 슬롯 무장 해제 + 보유 청산.

**리스크 스케일과 지배 규칙 (실험과 코어의 방화벽):**
- 백테스트 검증 슬롯(BRK_L)만 스케일 1.0. 나머지는 **0.2로 시작해 전방 트랙
  성적으로 사이즈를 번다.** 0.2는 V3 전 슬롯 가동 시 유니버스 MC 생존 게이트
  (95%p MDD ≤30%)를 지키는 최대 스케일(E13).
- **Upgrade 지배 규칙**: 검증 슬롯 신호는 같은 코인의 실험 포지션을 청산·승격
  (역방향 선점 불가). 실험이 코어를 잠식 못 하게 하는 기계적 방화벽.

### 3-0. FN/FP 관점의 필터 설계 (E14 — 신호 정밀도)

진입 신호는 "다음 24h가 비용 허들(0.4%) 이상 유리하게 움직이나?"의 이진 분류기다.
각 필터가 **False Positive**(진입했는데 틀림)와 **False Negative**(막았는데
승자였음)를 어떻게 맞바꾸는지 `scripts/fn_fp.py`로 측정한다. BTC 7일 돌파(raw)에
필터를 하나씩 얹은 결과:

| 필터 | 통과분 승률(=1−FP) | 거부분 승률(FN 비용) | 판정 |
|---|---|---|---|
| raw 돌파(무필터) | 0.55 | — | 기준 |
| + 1D 추세 정렬 | 0.56 | 0.54 | 값 있음 |
| + 4H 추세 정렬 | **0.57** | **0.37** | 강함 (거부분이 훨씬 못 이김) |
| + 거래량 ≥1.2× | 0.56 | 0.33 | 강함 |

각 필터가 FP를 낮추면서 거부한 진입(FN)은 실제로 덜 이긴다 → 필터가 값을 한다.
반대로 숏 슬롯(PBK_S FP 0.56, OSC_S FP 0.63)은 FP가 높아 실험 스케일 유지 근거.

### 3-1. BRK_L 상세 (백테스트 검증이 있는 유일한 슬롯)

다음 **전부** 충족 시 후보 (1H 봉 t 마감 기준):

```
① close(t) > max(high[t-168 .. t-1])          # 직전 7일 신고가 돌파
② 1D bias = up   (EMA20>EMA50 & close>EMA100) # 자산 자체 일봉 추세
③ 4H bias = up   (EMA20>EMA50)                # 중간 TF 정렬
④ BTC 1D 방향 = up                            # 광역 순풍
⑤ volume(t) ≥ 1.2 × median(volume, 48)        # 참여 확인
⑥ 국면 ∈ {TREND, RANGE}
```

**롱 전용인 이유(측정)**: 최근 16개월, 신저가 돌파 숏은 12/12 자산에서 진입 방향과
반대로 +3.1%/72h(숏스퀴즈 지배). 약세장은 관망이 포지션이다.

확신도: `conf = 0.55×base + 0.25×국면적합 + 0.20×TF정렬 + 펀딩수정자`
(base = 돌파마진·사전압축(BB폭 백분위)·돌파봉 실체·거래량의 평균,
펀딩수정자 = 숏 크라우딩(z<−1.5) +0.05 / 롱 크라우딩(z>+1.5) −0.10).
`conf ≥ 0.55`만 진입. **주의: conf는 진입 자격 필터일 뿐 사이징에 쓰지 않는다**
— 설계 측정에서 conf가 R을 랭킹하지 못했음(corr 0.005)을 그대로 반영한 결정.

### 4. 손절 배치 (E5 측정 반영)

```
SL = min( 돌파선 − 0.6×ATR1H,  close − 2.0×ATR1H )   # 더 먼 쪽
```
1H 노이즈 밴드(≈2×ATR) 안의 손절은 스윕당한다는 측정 결과로 변동성 하한 강제.
진입 후 손절은 타이트닝만 허용(절대 안 넓힘).

### 5. 사이징과 청산 봉쇄 (파산 확률 ≈ 0의 핵심)

```
stop_frac = (진입가 − SL) / 진입가
명목      = min( equity × 0.55% / stop_frac ,  equity × L )
증거금    = 명목 / L
L         = min( 티어 5x, 국면 캡, 자산 캡, L_liq )
L_liq     = 1 / (3×stop_frac + 1.5%)      # 청산 거리 ≥ 3×손절 거리 강제
```
- 손절 시 손실은 **항상 equity의 0.55%×(부스트 배수)** 로 상수 — 레버리지는 손익을
  키우는 게 아니라 증거금 효율·청산 거리를 정하는 수단(수학적 사실, BLUEPRINT §1).
- 청산가는 구조적으로 손절보다 3배 멀리. 엔진이 매 봉 이 불변식을 assert.
- r=0.55%는 몬테카를로 생존 게이트(1000경로, 95%p MDD ≤ 30%, P(MDD>50%)=0)를
  공격 레이어 포함 전 유니버스 트레이드로 통과하는 최대치.

### 6. 공격 레이어 — "좋은 기회에 크게" (V1.1, BTC 한정)

**핵심 원칙: 예측이 아니라 증거에 공격한다.** 어떤 트레이드가 대박일지 사전에
고르는 예측자(확신도 티어, 국면 origin 등)는 전부 측정에서 기각됐다(E9b·E10a).
살아남은 공격 방식은 두 가지 기계적 정책:

**① 피라미딩 (승자 증축)** — 포지션이 초기 손절폭 기준 **+2R, +4R**에 도달할
때마다 신규 리스크 유닛(0.55%)을 추가. 각 유닛은 현재 트레일 손절까지의 거리로
사이징되고 자체 청산안전 캡을 통과해야 한다. 최대 2회. 추세가 스스로 증명한
만큼만 노출이 커지고, 전체 포지션은 공유 트레일로 함께 청산된다.

**② 안티마틴게일 부스트 (승기 압박)** — 에쿼티가 고점 대비 2% 이내면 신규 진입
리스크 ×1.5 (0.55%→0.83%). 드로다운 −10%부터는 반감, −20%에서 신규 정지(기존
거버너). 시스템이 잘 맞는 국면에서 자동으로 크게, 어긋나는 국면에서 자동으로 작게.

**BTC 한정인 이유(정직)**: 전 유니버스에 켜면 MC 생존 게이트 붕괴(95%p MDD −53%,
수익도 악화) — 알트 챕에서 피라미딩은 고점 추격이 된다. BTC 단독 백테스트에서만
검증됐으므로(총수익 +19.5%→+33.9%, MDD −16.2%, 트리거·배율 ±20% 섭동 전부 양수
유지) BTC부터 단계 적용. 다른 자산 확대는 전방 트랙 증거가 쌓이면.

### 7. 청산 (측정으로 선택 — 부분익절·시간손절은 승자를 잘라 기각)

1. **초기 SL** (§4)
2. **8×ATR1H 샹들리에 트레일**: 진입 직후부터, `max(최고 종가 − 8×ATR, 현재 SL)`
3. **4H bias 플립**: EMA20<EMA50 2연속 4H봉 → 종가 청산 (추세 종료 판정)
4. **CRISIS 국면** → 즉시 청산
- 같은 봉에서 SL·상방이 겹치면 **항상 SL 먼저**(보수적 체결 가정).

### 8. 포트폴리오 가드

동시 ≤4 포지션(상관 경보 시 ≤2) · 코인당 1개 · 총 오픈리스크 ≤1.8% · 총 증거금
≤60% · 24h 실현손실 ≥4% → 24h 서킷 · 청산 후 12봉 쿨다운 · 유니버스 평균 상관
> 0.8 → 신규 반감.

### 9. 비용 모델

테이커 0.05%/side + 슬리피지 0.02~0.08%/side(자산군별, 체결가에 반영) + 실측 펀딩
(히스토리 없는 과거 구간은 8h당 0.01% 보수 드래그). 비용 2배 스트레스에서도
설계 구간 총수익 +74% 유지(G6).

### 10. 레버리지 프로파일 — 월 100% 목표의 정직한 스펙트럼

`PROFILE` 환경변수로 선택(기본 base). **프로파일은 사이징/공격 노브만 바꾸고
청산-안전 불변식(손절이 항상 청산보다 먼저)은 어느 프로파일에서도 불변.**

| 프로파일 | 리스크/트레이드 | 공격 범위 | 월 기하(설계) | **최고의 달** | 95%p MDD | **P(−50% DD)** |
|---|---|---|---|---|---|---|
| **base** (기본) | 0.5% | BTC만 | +2.6% | +41% | −28% | **0%** ✅ |
| turbo | 1.0% | 전 자산 | +2.8% | **+113%** | −76% | **58%** ⚠️ |
| max | 1.5% | 전 자산, 피라미드 3 | +4.7% | **+128%** | −87% | **86%** ☠️ |

**해석**: turbo/max는 강추세 달에 **실제로 월 +100%를 넘긴다** — 그게 레버리지의
힘이다. 그러나 같은 레버리지가 파산성 드로다운(−50%+) 확률을 58~86%로 만든다.
즉 월 100%는 "가끔 일어나는 사건"으로는 가능하지만 "매달의 기대값"으로는 계좌를
반드시 0으로 보낸다. **base로 살아남으며 복리하고, 강추세라는 확신이 설 때만
수동으로 turbo를 켜는 것**이 이 스펙트럼의 정직한 사용법이다(라이브는 여전히 G8
게이트 뒤에 잠겨 있음). max는 페이퍼 전용 가드레일.

```bash
PROFILE=base  python3 run_live.py    # 생존 (기본)
PROFILE=turbo python3 run_live.py    # 공격 — 파산 확률 58% 인지하고
```
