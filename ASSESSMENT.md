# ASSESSMENT — v3 완성도 평가 및 ROI(30d) 50%+ 로드맵 (2026-07)

템포 결정 포함 시스템 전반 점검. 방법론: 모든 실험은 2025 설계 구간에서만 수행,
2026 홀드아웃은 선택에 사용하지 않음. 과적합 경계를 위해 **이번 점검에서 시도한
모든 반증 실험을 성공/실패 무관하게 기록**한다.

4H/8H/1D가 동급이라는 것은 시스템이 템포에 강건하다는 좋은 신호이며, 설계 구간

8가지 반증 시도에서 동결된 v3 구성이 전부 살아남았다. **좋은 시스템의 증거는
"더 좋은 변형을 찾았다"가 아니라 "무너뜨리려는 시도가 실패했다"이다.**

### 정직한 약점 (증거 기반)

1. **HYPE 집중 리스크 (가장 중요)**: BTC/ETH/SOL만의 XS는 −1.96%다. 현재 엣지의
   대부분은 HYPE의 특이성(타 자산과 상관 0.58 vs 메이저 블록 내부 0.70)에서 나온다.
   HYPE가 블록에 동화되면 엣지는 감쇠한다. → §4 로드맵 (a)가 이것의 구조적 해법.
2. **표본 1개**: 18.5개월, 단일 시장 체제 사이클. 게이트는 강건성을 말할 뿐
   미래 수익률을 보증하지 않는다.
3. **2025Q1 −5.0%**: HYPE 상장 전 3자산 구간은 얇은 횡단면(3종 순위)의 약점을 보여준다.
   `xs_min_assets=3`은 하한이지 최적이 아니다.
4. **펀딩 데이터 3개월**: OKX API 한계로 초기 15개월의 펀딩 비용이 시뮬레이션에서
   과소 계상 (XS는 준시장중립이라 순펀딩이 작지만, 0은 아님 — 실측 −243 USD/기간).

## 3. 유니버스 확장 실험의 교훈 (개선 아이디어의 방향타)

XRP/DOGE/ADA/LINK/AVAX/LTC/DOT/TRX/BCH — "2025년 이전 상장 + 상위 유동성"이라는
사전 기준으로 9종을 추가했으나 결과는 파괴적이었다(−7.2%). 원인은 측정으로 확정:
이들의 평균 쌍상관은 0.69로 **한 덩어리 블록**이며, 상관된 자산들 사이의 상대강도
순위는 신호가 아니라 노이즈다. 잔차 모멘텀으로 베타를 벗겨도 구제되지 않았다
(구형 메이저의 특이 성분은 추세가 아니라 평균회귀 성향).

**교훈: XS의 스케일 축은 "자산 개수"가 아니라 "분산(dispersion)"이다.** HYPE가
가치 있었던 이유는 신규 상장 자산 특유의 낮은 상관·특이 플로우였다. 이것이 §4의
로드맵을 결정한다.

## 4. ROI(30d) 50%+ — 정직한 산술과 구조적 로드맵

**현재 위치**: 월평균 +2.1% (vb 0.2), 최고의 달 +11.7%, vb 0.6 공격 설정 시 월평균
+3.5%/MDD −17%. **월 50%는 현재 시스템의 ~15배**다. 샤프를 고정한 채 변동성만
8배로 올리는 것(vol_budget ~1.6)은 MC 95% MDD가 −25% 한도를 뚫는 파산 영역이다.
따라서 경로는 리스크 증가가 아니라 **신호 밀도·질의 구조적 개선**뿐이며, 각 항목의
기대 기여와 과적합 위험을 명시한다:

| # | 개선 | 메커니즘 | 기대 기여 | 과적합 위험 |
|---|---|---|---|---|
| (a) | **상장 인지형 신규자산 편입 룰** | "상장 후 6~18개월 + 유동성 기준 통과" 자산을 자동 편입/졸업시키는 롤링 룰. HYPE-류 특이 자산을 상시 2~4종 유지 → 분산의 원천을 시스템화 | 월 +1~3%p (추정) | **중** — 과거 상장 목록 복원 시 생존 편향 주의. 백테스트보다 페이퍼 전방 검증이 적합 |
| (b) | vol_budget 0.2→0.4 단계 상향 | 프론티어 실측상 월 +2.1→+2.9% (샤프 완만 감쇠) | 월 +0.8%p | **저** — 이미 실측된 프론티어. 조건: 페이퍼 3개월 확인 |
| (c) | 메이커 집행 전환 | 리밸런스 지정가화: 수수료 5→2bp. 비용 스트레스 실측(G3)이 비용 탄력성을 증명 (연 수수료 4.2k→~1.7k) | 월 +0.2~0.3%p | **저** — 미체결 리스크만 관리 |
| (d) | 펀딩 캐리 슬리브 | 데이터 축적(90일+) 후 |F_z| 기반 캐리 수확. XS와 이론상 무상관 | 월 +0.5~1%p (미검증) | **중** — 지금은 검증 불가, 데이터가 쌓이면 설계 구간 신설 |
| (e) | 포트폴리오 레벨 vol 타게팅 | 자산별 예산의 합이 아니라 포트폴리오 실현 vol을 직접 타게팅 → 상관 급등 구간 자동 감속 | MDD 개선 (수익 아님) | **저** |
| (f) | XS 멀티 지평 앙상블 (60+120+180) | G2에서 xs_look±20% 전부 흑자 → 지평 평균화는 튜닝이 아니라 강건화 | 분산 감소 | **중하** — 채택 시 설계 구간 재검증 필요 |

**현실적 궤적**: (b)+(c)는 즉시 실행 가능하고 실측 근거가 있어 월 ~3.5~4%. (a)가
성공적으로 특이 자산 2~4종을 추가 공급하면 월 5~8% 영역이 시야에 들어온다.
**월 50%는 이 로드맵의 종점이 아니라 방향이다** — 그 목표를 백테스트로 "보여주는"
방법은 과적합뿐이라는 사실은 13자산 실험(설계 +40%였던 v2-CHOP의 홀드아웃 붕괴,
이번 −7.2%)이 반복 실증했다. 파산 확률을 0 근처에 묶은 채 기하평균을 올리는 것,
그것이 이 시스템이 목표에 접근하는 유일한 문이다.

## 5. 즉시 조치된 디테일 개선 (이번 점검에서 코드 반영)

- **합집합 인덱스 엔진**: 늦게 상장된 자산이 전체 패널 히스토리를 자르던 구조적
  결함 수리 (유니버스 확장의 전제조건). 상장 전 구간은 NaN 게이팅으로 자동 제외.
- **자산별 안전 기본값**: 레버리지 캡/베타/슬리피지 사전에 없는 자산은 보수적
  기본값 (1.0x / 1.3 / 8bp) 적용 — 신규 자산 추가가 KeyError로 죽지 않음.
- **얇은 횡단면 가드**: 생존 자산 < 3이면 XS 신호 자체를 차단.
- **템포 파라미터화** (`bars_per_year`): 모든 연환산·통계가 템포 독립적.
- **잔차 모멘텀 옵션** (`xs_residual`, 기본 off): 기각 기록과 함께 보존 —
  미래 유니버스에서 재검정할 수 있는 계측 장치.
