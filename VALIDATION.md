## 정직한 종합 판정

**라이브 배포: 불가.** G3(홀드아웃 기대값)과 G4(워크포워드)가 명시 기준을 실패했다.

같이 봐야 할 사실들:
- 홀드아웃 반년은 추세 부재/약세 구간이었고, 롱 전용 추세 시스템은 설계 의도대로
  **거의 진입하지 않고(월 ~3건) 자본을 보존했다**(−1.6%, MDD −6.7%). 생존 기계는
  작동한다. 그러나 수익 엣지의 전방 증거는 없다. n=17로는 어느 쪽도 증명 불가.
- G4 실패는 추세추종의 구조적 특성(수익의 시간 집중)이지만, 기준을 사후에 바꾸지
  않는다 — 실패는 실패다.
- 이 시스템의 현실적 기대치: 강한 추세 국면에서 월 +10~30%대(설계 구간 최고 2개월
  +70%), 무추세 구간에서 0 ± 소폭. **"매달 100%"는 이 데이터에서 존재하지 않았다.**
  월 100%를 강제하는 파라미터는 전부 MC 파산 게이트를 실패한다(r≥1%도 이미 실패).

## 재현 방법

```bash
python3 -m pytest tests/ -q                      # G1
python3 run_backtest.py 2023-06-01 2026-01-31    # G2
python3 - <<'PY'                                  # G4~G7
from fabletradebot.config import Params
from fabletradebot.validation import *
from fabletradebot.backtest import run_backtest
p = Params()
print(walk_forward('data', p, '2023-06-01', '2026-01-31'))
print(sensitivity('data', p, '2023-06-01', '2026-01-31'))
print(cost_stress('data', p, '2023-06-01', '2026-01-31'))
print(monte_carlo(run_backtest('data', p, start='2023-06-01', end='2026-01-31')['trades']))
PY
```

홀드아웃 재실행은 금지 대상이 아니라 "재조회 시 공개" 대상이다: 이 문서 이후
홀드아웃 구간을 다시 돌린 실험이 있으면 여기 기록한다. (현재: 최초 1회뿐.)
