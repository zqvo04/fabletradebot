# EXIT_REDESIGN — 횡보 정체 포지션 청산 개선 설계 (V6 후보, 사전 등록)

> 상태: **X-A 측정 완료 — 기각.** 설계 구간 whale 리플레이(2023-06~2026-01,
> 반쪽 교차, `scripts/exp_stall.py`)에서 stall_bars {18,24,36} × stall_trail
> {2.5,3,4} 9셀 전부 터미널 부 붕괴(baseline 67.5x → 0.8~12.1x, 최고셀도 −82%),
> MDD 8/9셀 악화(−56~−76% vs −48.9%), 격자 비단조(ret 요동 = G5 경로 노이즈).
> 근본 원인: X-A의 러너-안전 논리("러너는 best_close 갱신 → 안 조여짐")가 실증
> 반증됨 — 트레이드 수 364→445~551(+22~51%)은 조임이 추세 중간 눌림에서 대량
> 발화했다는 직접 증거. E9/E16의 "승자 조기 절단 → 복리 붕괴"가 정체 조건부로
> 위장돼 재현. **X-A 실패 → §2 규율대로 X-C가 다음 후보로 활성.** base 불변,
> config 미변경(스위치 off 유지), pytest 69/69.

작성 계기: 2026-07-15 진입한 AVAX PBK_S(whale)가 +0.97R 피크(6.447 저점 부근)를
찍고도 은행하지 못한 채 74+시간 횡보 속에 좌석을 점유. whale은 단일 좌석이므로
정체 포지션의 기회비용이 포트폴리오 모드와 비교 불가하게 크다.

원칙: 여기 있는 것은 **설계와 측정 계획**이지 채택이 아니다. 모든 후보는 설계
구간 whale 백테스트에서 G5 게이트(반쪽 부호 일관성)를 통과해야만 config에 들어간다.
E9/E16이 이미 기각한 형태(무조건 부분 TP, 무조건 giveback)는 재도입하지 않는다.

## 1. 진단 — 현행 청산 세트에 "정체(stagnation)" 축이 없다

현행 트렌드 슬롯(BRK/PBK/RCL) 청산 경로 전수 (engine.py §manage):

| 경로 | 발동 조건 | AVAX 사례에서 왜 안 걸렸나 |
|---|---|---|
| SL/Trail | 가격이 스탑 도달 | 트레일 8 ATR 샹들리에 — best_close 6.447 기준 트레일이 초기 스탑 근처(≈6.75+). 횡보에서는 영원히 안 조여짐 |
| TP | tp1 > 0 | 트렌드 슬롯은 tp_r=0 (E9: 부분 TP가 승자를 자름 — 근거 있는 선택) |
| SignalFade (수익 은행) | ① giveback 레그: peak≥1R 후 반납 — **E16에서 비활성화(hold_giveback=1.0)** ② conviction-collapse: hold_conf<0.50 ×2봉 | ①은 꺼져 있고 peak 0.97R로 arming(1.0)에도 미달. ②는 hold_conf 0.77 — 완만한 TREND_DOWN 정렬이 유지되는 횡보에서는 절대 안 떨어짐 |
| LossFade | 손실 중 + conviction<0.50 ×2봉 | 수익 중이라 해당 없음 |
| BiasFlip | 4H 바이어스 역전 ×2봉 | 횡보는 바이어스를 뒤집지 않음 (래깅) |
| Timeout | time_stop_bars>0 | 트렌드 슬롯은 0 (E9) |
| Regime | CRISIS | 해당 없음 |

구조적 결론: 모든 청산이 **가격 역행**(SL/Trail/BiasFlip) 아니면 **논지 붕괴**
(Fade류)에만 반응한다. "가격도 논지도 무너지지 않았지만 더 가지도 않는" 횡보
정체에 반응하는 축이 하나도 없다. 이것이 사용자 관찰("최저점에서 익절 못하고
너무 오래 끌고 감")의 정확한 메커니즘이며, 파라미터 미세조정이 아니라 축 하나가
빠진 문제다.

주의(정직성): E9·E16의 기각은 유효하다 — 시간컷/giveback을 **무조건** 걸면 추세
승자를 자른다. 하지만 두 측정 모두 "정체 조건부" 변형은 측정한 적이 없고, E16은
whale에서 giveback을 껐을 뿐 정체 축의 대체물을 넣지 않았다. 아래 후보들은 전부
"러너는 건드리지 않음"을 구조적으로 보장하는 조건부 형태다.

## 2. 사전 등록 후보 (각각 독립 측정, a-priori 파라미터)

### X-A — 정체 조임 트레일 (stall-tightened chandelier) · **기각(측정 열위, 상단 요약)**
best_close 갱신이 `stall_bars`(24 = 1일) 동안 없고 peak_r ≥ 0.5 이면, 그 포지션의
트레일 폭을 8 → `stall_trail_atr`(3.0, E9 초안에서 쓰던 표준폭) ATR로 조인다
(포지션별 원웨이 래칫: 한 번 조여지면 이후 신규 best_close가 나와도 8로 되돌리지 않는다).
- 러너 보호가 구조적: 달리는 추세는 계속 best_close를 갱신하므로 절대 안 조여짐.
  정확히 실패 모드(정체)에만 조건화 → 과적합 표면이 가장 작다.
- 파라미터 3개 모두 a-priori: 24봉(1일), 3 ATR(교과서 샹들리에), peak 0.5R
  (손실 포지션은 LossFade/SL 관할 — 이 축은 수익 은행 전용).
- 강건성 스윕: stall_bars {18,24,36} × stall_trail {2.5,3,4} — 최댓값 고르기가
  아니라 평탄성 확인용. 부호가 셀마다 뒤집히면 기각.

### X-B — 트렌드 슬롯 시간컷 재측정 (whale 한정)
기존 배관 그대로 `time_stop_bars`=96(4일), `time_stop_min_r`=0.3을 whale 프로파일에만
켠다. E9의 기각은 포트폴리오 BRK_L 기준 — whale의 좌석 기회비용은 그 측정에 없던
항이므로 "재측정할 자격"이 있는 것이지 채택 근거가 있는 게 아니다. 반쪽 중 한쪽이라도
터미널 자산이 악화되면 즉시 기각. 스윕 {72,96,120}은 평탄성 확인용.

### X-C — 모멘텀 조건부 giveback 재무장 · 낮은 사전확률
E16이 기각한 giveback을 "4H 모멘텀 소진(mom ≤ 0) 동안만 무장"으로 조건화. X-A와
역할이 겹치고 조건이 하나 더 붙으므로(파라미터+1) X-A가 게이트를 통과하면 측정하지
않는다(중복 축 금지). X-A가 실패할 때만 측정.

### 기각 확정 (측정 불요)
- 트렌드 슬롯 무조건 고정 TP: E9 직접 반증. 재론 없음. ("TP가 없다"는 관찰은
  맞지만, 없는 것이 측정된 선택이다 — 문제는 TP 부재가 아니라 정체 축 부재.)
- hold_conf_exit 상향(0.50→0.6대): AVAX 사례의 hold_conf가 0.77 — 횡보에서도
  정렬 점수가 높게 유지되므로 임계 상향으로는 못 잡고, 올리면 E17에서 측정한
  conviction-collapse의 정상 작동만 훼손한다.

## 3. 측정 프로토콜 (전 후보 공통)
1. 설계 구간 whale 백테스트, 변경 전 이 문서로 사전 등록 완료 상태에서만 실행.
2. 게이트: 터미널 자산·MDD·avg_r 이 **두 반쪽 모두** 현행 대비 악화 없음(G5).
   한쪽 개선·한쪽 악화 = 경로 노이즈로 기각 (E19 선례).
3. base 프로파일 회귀: X-B는 whale 전용 배선이라 base 불변이 자명해야 하고,
   X-A는 base에서도 켜지므로 base 지표 악화 시 whale-전용으로 강등 후 재측정.
4. MC 생존 게이트(95%p MDD ≤ 30%, base) 재확인 + pytest 전수 녹색.
5. 채택 시에도 risk 변경 없음 — 청산 축 추가는 사이징/레버리지/liq-invariant를
   건드리지 않는다.

## 4. 부가 관찰 (이번 설계 범위 밖, 기록만)
- peak_r 0.97 vs arming 1.0: giveback 레그가 꺼진 현재는 무의미한 경계지만,
  X-C를 측정하게 되면 arming 경계 민감도(0.7/1.0)를 같이 봐야 한다.
- 라이브 AVAX 포지션 자체는 이 설계로 소급 처리하지 않는다 — 채택된 로직이
  다음 재시작부터 관리한다.
