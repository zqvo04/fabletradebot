## 1. 목표 정직성 — "평균 월수익 100%"의 산수

측정된 현재 위치:
- 설계 구간 whale (E17 최종): +5,311% / 32개월 → **기하평균 ~13.3%/월** (인샘플).
- 전방 페이퍼 (07-12 앵커~): 10,000 → 8,864 (**−11.4%**), 고점 10,062 대비 −11.9%.
- G8: 전방 샤프 ≥ 백테스트 × 0.5 미달 시 라이브 금지 — 현재 전방은 인샘플에 미달.

월 100%는 인샘플 최고 측정치의 **7.5배**다. full-margin whale에서 트레이드당
계좌 손익 ≈ L × 가격변동이므로, 5x 티어·stop 2.6% 기준 −1R ≈ −13%,
+2R ≈ +26% equity. ln2 ≈ 0.69를 채우려면 대략 **월 6트레이드에서 승률 ~67%,
평균 승자 +2R** (4×ln1.26 − 2×|ln0.87| ≈ 0.66)이 필요하다.

결론: 목표를 좁히는 병목은 **사이즈가 아니라 A급 다리(leg)의 월간 포착 횟수**다.
사이즈를 올리는 접근(레버리지·리스크 확대)은 CV-C에서 MDD −48→−65%로 이미
기각됐고, 생존 게이트(§0 BLUEPRINT)와 정면 충돌한다. 따라서 본 설계는
**"생존 게이트 안에서 자격 신호 포착 용량을 늘린다"**로 목표를 재정의한다.
월 100%를 하드 타깃으로 삼아 리스크를 역산하는 순간이 곧 파산 경로 설계다 —
이 문서는 그 접근을 §3에서 명시적으로 기각한다.

## 2. 사전 등록 변형 (측정 전 — 채택/기각 조건 포함)

공통 게이트 (E17/E18과 동일 규율):
- 설계 구간(2023-06-01~2026-01-31) 리플레이 + **반쪽 교차**(H1/H2 부호 일치 필수)
- G5(신규 파라미터 ±20% 부호 유지) · G6(비용 2배) · G7(MC 부트스트랩)
- **base 프로파일 바이트 불변** (whale 전용 변경, 포트폴리오 모드 비오염)
- 홀드아웃 미개봉. 전방 페이퍼가 최종 심판.
- pytest 전체 녹색 + liq>SL 불변식 유지.

### SR-A — whale-N: 좌석 수 N ∈ {1, 2, 3} (주 변형, 최대 레버)

- **기계적 정의**: `max_positions=N`, 좌석당 margin_frac 1/N (full-margin 균등
  분할), `max_positions_corr=max(1, N-1)` (기존 상관 경보 기계 재사용 — 유니버스
  30D 페어와이즈 상관 > 0.8 시 좌석 축소). 그 외 전부 현행 whale.
- **a-priori 근거 (양방향 정직 기술)**:
  - (+) §0의 측정: 좌석 1개가 38시간 동안 16건을 폐기. 독립적 양의 기대값
    신호가 월 n건일 때 포착률은 좌석 수에 비례(포화 전까지).
  - (−) 승자당 복리 집중이 1/N로 희석 — whale의 tail upside 논리 자체가 약화.
    crypto 상관 ~0.8에서 N좌석은 종종 같은 베팅의 분할일 뿐이다(분산 이득 환상).
    E10c 선례: 공격 레이어의 유니버스 확대는 MC 생존 게이트를 깨뜨렸다.
- **판정 규칙**: N* 채택 조건 = 설계 구간 기하수익 ≥ N=1 **이고** MC 게이트
  (95%p MDD, P(>50%)) 비열위 **이고** 반쪽 부호 일치. 하나라도 미달 → N=1 유지.
- **예상 기각 시나리오**: 상관 장세에서 N=2가 사실상 같은 방향 2배 노출이 되어
  MDD가 악화되면 기각. 이 경우 결론은 "좌석 수가 아니라 신호 커버리지가 병목".

### SR-B — dead-seat 회전: 죽은 좌석만 도전 허용 (소 변형)

- **기계적 정의**: 코인 간 displacement를 **보유 포지션이 이미 fade-armed 상태**
  (`loss_fade_streak ≥ 1` 또는 `fade_streak ≥ 1` — 즉 다음 봉 청산이 예약된
  상태)일 때만 허용. 도전자는 보유자가 나가는 같은 봉에 좌석을 채운다(현행은
  청산 봉 + 다음 신호 대기 지연). 도전자 간 우선순위는 기존 fill_order
  (proven 우선 → conf) 그대로.
- **a-priori 근거**: 건강한 승자는 절대 건드리지 않으므로 E16-안전. conf를
  코인 간 좌석 비교에 쓰지 않으므로 E9-안전. 순수 지연 제거(청산→재충원
  레이턴시 1~수 봉 단축).
- **정직 공개**: 이 변형은 **HYPE 사건을 해결하지 못한다** (AVAX는 건강했다).
  사건에 규칙을 맞추지 않았다는 증거로 명시해 둔다.
- **판정 규칙**: 설계 구간에서 기대값·MDD 비열위 + 반쪽 일치. 개선이 노이즈
  수준(<+0.01R)이면 복잡도 비용으로 기각(단순성 우선).

### SR-C — 사유별 쿨다운: 손실 청산만 12봉, 수익 청산은 2봉 (소 변형)

- **기계적 정의**: `cooldown_bars` 적용을 청산 사유로 분기 — SL/LossFade
  (실현손실) 후 12봉 유지(설계 근거: "RSI<20 지속, 칼날 재진입 방지" — 손실
  케이스의 근거), 트레일/SignalFade/Upgrade(실현이익) 후 2봉. 재진입은 여전히
  전체 게이트를 통과한 신규 신호 필수.
- **a-priori 근거**: 문서화된 쿨다운 근거(칼날)는 수익 청산에 적용되지 않는다.
  수익 트레일 청산 후 추세 재개 시 재승차 지연이 순수 비용.
- **정직 공개**: HYPE 07-15 03:00 건은 5봉 내 청산(손실 추정, pre-history로
  사유 미복구) — 손실이었다면 이 변형도 HYPE 사건을 바꾸지 못했다.
- **판정 규칙**: SR-B와 동일 + 신규 파라미터(2봉)에 G5 ±20%… 봉 단위 이산이므로
  {1, 2, 3}봉에서 부호 유지 확인.

### SR-D — 숏 계열 승급 프로토콜 명문화 (거버넌스 — 측정 불요, 즉시 채택 가능)

현행: 숏 전 슬롯 `risk_scale 0.20` 영구 고정(수동 개입 전까지). E12가 "전방
승급제"를 선언했으나 승급 조건이 수치로 존재하지 않는다 — 이 상태로는 SR-A/B가
통과해도 숏 좌석은 영원히 0.20 스케일이고, 코인 내 Upgrade 자격(1.0)도 영원히
없다.

- **승급**: 슬롯별 전방 실거래 n ≥ 30 & 비용 후 기대값 > +0.05R (G2 문턱 재사용,
  신규 적합 아님) → 0.20 → 0.50. n ≥ 60 & 유지 → 1.0 (proven, Upgrade 자격).
- **강등**: rolling-30 기대값 < 0 → 한 단계 강등. 0.20 미만으로는 내리지 않음
  (증거 수집 자체가 목적 — E12 원리 유지).
- **금지**: 설계 구간 재적합으로의 승급(E12/E18 I5 규율). 전방 증거만 유효.
- 판정 주체: 매주 채점 리포트. 수치는 전부 기존 게이트(G2)에서 상속 — 시장에
  적합시킨 자유 파라미터 없음.

## 4. 실행 순서와 검증 프로토콜

```
1. SR-D 명문화 → 코드/문서 반영          → verify: pytest + 채점 리포트에 승급 카운터
2. SR-A 리플레이 (N=1/2/3 × H1/H2)       → verify: §2 공통 게이트 전부
3. SR-B, SR-C 리플레이 (SR-A 승자 위에)   → verify: 동일 + 단순성 문턱(+0.01R)
4. 통과 변형만 config 반영, E19로 기록    → verify: base 바이트 불변 diff + pytest
5. 전방 페이퍼 재개 — G8이 최종 판정
```

리플레이 명령: `run_backtest(data_dir, p=replace(profile("whale"), ...), start, end)`.
데이터 스냅샷은 2026-07-11까지 — 설계 구간 리플레이에는 충분(홀드아웃 불침).

## 5. 예상 효과의 정직한 한계

- 전 변형이 통과해도 이것은 **포착 용량** 개선이지 엣지 창출이 아니다. 인샘플
