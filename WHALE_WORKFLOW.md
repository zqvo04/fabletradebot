# WHALE_WORKFLOW — 좌석 유지/회전 워크플로우 정밀 진단 (V5.1, 측정 전 사전 등록)

전제: V5(E17)에서 conf(CV-A, whale 제외)·hold_conf(HV-A)가 개선됨. 이 문서는 그
개선된 점수들이 whale 단일 좌석의 **점유 → 유지 → 청산 → 회전** 로직과 만나는
전 구간을 코드 추적 + 설계 구간 실측(2023-06~2026-01, n=411, 반쪽 교차)으로
진단한다. 홀드아웃 미개봉.

---

## 1. 워크플로우 전 구간 추적 (코드 레벨)

매 1H 봉 t에서:

1. **점유(fill, engine.py §1)**: `pendings`를 `(risk_scale, conf)` 내림차순 정렬
   (E15: 검증 슬롯이 실험 슬롯 conf에 절대 안 밀림). 단일 좌석이므로 정렬 1등만
   실제 점유 가능.
   - 좌석이 비었으면(len==0) 정렬 1등이 full-margin·conf→레버리지 티어로 점유.
   - 좌석이 같은 코인으로 차 있으면 **Upgrade**: 검증(1.0)이 실험(<1.0)을 청산·승격.
     **교차 코인 승격은 없음** — 다른 코인의 검증 신호는 좌석을 못 뺏음(E16 W-A로
     측정·기각: 실험 포지션을 중도 청산하면 승자가 될 것도 잘라 순손해).
2. **유지(manage, §2)**: 매 봉 hold_conf(HV-A: 연속 fit + 대칭 rsi) 재계산.
   스탑/트레일/펀딩/피라미딩(whale은 off) 처리.
3. **청산(§2 exit legs, 우선순위)**: Regime(CRISIS) → SignalFade(수익 보호,
   conviction-collapse만; giveback off) → LossFade(손실 조기 컷) → BiasFlip →
   Timeout. 그 외 SL/Trail/TP는 인트라바.
4. **회전(§3 + cooldown)**: 청산 후 코인당 12봉 쿨다운(`exp` 플래그: 실험발
   쿨다운은 검증 후보를 막지 않음). 쿨다운 후 다음 정렬 1등이 좌석 재점유.

## 2. 실측 누수 진단 (설계 구간 whale n=411)

### 2a. 좌석 배분이 엣지와 정반대로 왜곡 (구조적)

| 그룹 | 좌석수(n) | 비중 | sumR | meanR |
|---|---|---|---|---|
| 검증 BRK_L | 61 | **15%** | **+73.5** | +1.205 |
| 실험 슬롯 전체 | 350 | **85%** | +6.9 | +0.020 |

**BRK_L이 총엣지의 ~91%를 15% 좌석-시간에 생산**하고, 실험 슬롯 85%는 사실상
브레이크이븐으로 좌석을 점유한다. 최악: BRK_S −15.7R(n32), RCL_S −5.9(n57),
PBK_L −2.6(n84). 유일 실험 승자 PBK_S +26.9(n118)는 숏(E13/E14의 숏 약세와
국면 의존성 주의). 원인: BRK_L은 TREND_UP/RANGE + 신선한 7d 돌파에서만 무장 →
드물게 발화하는데, 실험 슬롯(눌림·숏·재탈환)은 매 봉 다발 발화해 좌석을 상시 점유.

### 2b. 교차 코인 차단 — BRK_L 기회의 29%가 실험 좌석에 막힘

BRK_L 후보 발생 봉 985개 중: 좌석 비어있음(체결 가능) **0**, BRK_L이 이미 점유
703, **실험 슬롯이 점유해 차단 282 (29%)**. 단일 좌석 + 교차코인 승격 부재의
결과. 단 — E16이 교차코인 승격(W-A)을 이미 측정·기각(승자 될 실험 포지션 강제
청산이 더 비쌈)했으므로, 이 29%는 **버그가 아니라 측정된 트레이드오프**다.
공략은 좌석을 뺏는 게 아니라(§3) 실험이 애초에 좌석을 덜 잡게 하는 진입 측.

### 2c. 진입/홀드 비정합 — 좌석이 자기 홀드 로직이 거부할 포지션을 받는다 ★핵심

| 측정 | 값 |
|---|---|
| SignalFade 트레이드의 **진입시점** hold_conf | 평균 **0.405** (median 0.410) |
| SignalFade 발화 시점 | **21/23이 정확히 2봉**(= hold_conf_bars) |
| 전체 whale 트레이드 진입시점 hold_conf | 평균 0.720 |
| **승자**(R>0) 진입 hold_conf vs **패자** | 0.679 vs **0.740** (패자가 더 높음) |
| corr(진입 hold_conf, R) | **+0.034** (예측력 없음) |

**개선된 hold_conf가 청산(exit)에는 쓰이지만 진입(entry)에는 안 쓰인다.** 그래서
좌석은 hold_conf가 이미 0.50 바닥 밑(0.405)인 포지션을 받아들이고, conviction-
collapse 청산이 2봉 뒤 곧바로 내보낸다 — **스스로 유발한 회전**(진입 로직과 홀드
로직이 불일치). 이 루프를 닫는 것이 개선된 conf/hold_conf → 좌석 로직으로
이어지는 워크플로우의 빠진 고리.

### 2d. 회전 품질 (참고)

청산 직후 다음 트레이드 meanR: SignalFade 후 **−0.067**, Trail 후 −0.025 —
좌석이 평범한 실험 신호로 회전(2a의 발현). LossFade 후 +0.766(손실 컷이 좌석을
승자에 넘김 — 건강), BiasFlip 후 +0.252. SignalFade/Trail 후 회전이 약한 음수인
것은 2a·2c의 하류 증상.

## 3. 측정으로 반전된 가설 (정직 — "누수처럼 보였으나 load-bearing")

- **쿨다운은 누수가 아니다.** cd 12→6→0: ret 53x→38x→21.5x, MDD −46.5%→−57.2%로
  급격 악화. 단일 좌석에서 쿨다운은 knife-catch/회전을 막는 보호 장치 → **cd=12
  유지**. (conf 티어 de-risking이 load-bearing이던 것과 같은 패턴.)
- **교차코인 승격 재개는 안 한다.** E16 W-A 기각(승자 될 실험 강제청산)이 여전히
  유효. 2b의 29% 차단은 진입 측(§4)에서 간접 완화.

## 4. 사전 등록 수정 (E17 프로토콜과 동일 게이트)

### WF-A (채택 후보 — ★): 진입/홀드 정합 게이트

개선된 hold_conf를 **진입에도** 적용: 후보의 held-side hold_conf(결정봉 값)가
청산 바닥 미만이면 좌석 후보에서 제외. **문턱 = hold_conf_exit(0.50) 재사용 —
청산과 문자 그대로 동일 문턱, 신규 파라미터 0개(제로-DOF).** 무장 조건은
hold_conf_exit>0 (whale만; base는 청산 비무장이라 자동 불변).

측정 (whale, 설계 구간, 반쪽 교차):

| floor | n | exp | ret | MDD | H1 / H2 |
|---|---|---|---|---|---|
| 없음(현행) | 411 | +0.1955R | 53.1x | −46.5% | +0.254/+0.150 |
| 0.45 | 388 | +0.2229R | 61.8x | −48.5% | +0.277/+0.184 |
| **0.50 (= hold_conf_exit)** | **364** | **+0.2102R** | **67.5x** | −48.9% | **+0.223/+0.200** |
| 0.55 | 340 | +0.2347R | 68.6x | −49.2% | +0.236/+0.234 |

- floor 0.40~0.55 전 구간 exp·ret·양쪽 반기 개선(단조에 가까움) → **강건**.
- 제로-DOF 선택 = floor=hold_conf_exit(0.50): ret 53→67.5x(+27%), exp +0.196→
  +0.210R, 양쪽 반기 상승, MDD 소폭 비용(−46.5→−48.9%). 문턱을 0.55로 올리면
  더 좋으나 그건 스윕 최적화라 채택 안 함(과적합 경계) — 청산과 묶인 0.50만.
- 부수효과: 실험 슬롯의 즉시-붕괴 진입이 걸러져 2a 좌석 왜곡·2b 교차 차단을
  **진입 측에서 완화**(좌석을 뺏지 않으므로 W-A 회전 문제 없음).
- 게이트: 반쪽 부호 안정(양쪽↑ ✓), base 회귀 불변(무장 조건으로 자동 ✓), G5
  floor 민감도(0.45/0.50/0.55 전부 개선 ✓). whale은 눈 뜨고 선택 프로파일이라
  생존 MC 비대상이나 MDD 변화 보고.

### 기각/보류 (측정 또는 근거)
- cooldown 완화: 기각(§3, load-bearing).
- 교차코인 승격: 기각(E16 W-A 유효).
- 실험 슬롯 진입 문턱 상향 등 2차 항목: WF-A 전방 확인 후 재검토(신규 파라미터
  최소화).

## 5. 요약 (진단)

개선된 hold_conf(HV-A)는 좌석 **유지·청산**에는 반영됐으나 **점유(진입)**에는
빠져 있었다 — 좌석이 hold_conf 0.405짜리 포지션을 받아 2봉 만에 conviction-
collapse로 회전시키는 자기유발 churn이 실측(진입 hold_conf가 R을 예측 못하고,
패자가 승자보다 진입 conviction이 오히려 높음). 빠진 고리는 **진입/홀드 정합
게이트(WF-A)** 하나이며, 청산과 동일한 0.50 문턱을 재사용하는 제로-DOF 수정으로
설계 구간 whale ret +27%·양쪽 반기 개선을 낸다. 쿨다운·교차코인 승격은 누수처럼
보였으나 측정상 load-bearing이라 건드리지 않는다.

## 6. 개선 플랜 (V5.1 사전 등록) — 워크플로우 논리 연결의 완성

원칙은 "하나의 판단 체계": 진입·유지·청산·회전이 **같은 conviction 척도를 같은
문턱으로** 읽어야 한다. 아래 5개 정합 불변식(I1~I5)이 설계 목표이고, 각 단계는
불변식 하나를 복원한다. 신규 파라미터 총 0개(전부 기존 값 재사용), 판정 게이트는
E17과 동일(반쪽 교차 부호 안정 + base 비무장 경로 불변 + G5/G6/MC 보고 + 전방
페이퍼 재검증). 변형은 하나씩 스택, 채택분만 남긴다.

### 정합 불변식과 현재 위반 지점

| # | 불변식 | 현재 위반 (측정) | 단계 |
|---|---|---|---|
| I1 | 좌석은 홀드 로직이 즉시 거부할 포지션을 받지 않는다 | SignalFade 진입시 hold 0.405, 21/23이 2봉 청산 (§2c) | WF-A |
| I2 | 상위 정보(연속 conviction)가 하위 정보(이진 선조)에 기각당하지 않는다 | BiasFlip 45건 중 44건이 hold≥0.50("논지 생존", 평균 0.544)에서 발화, 컷 후 48h 트레이드 방향 +2.07% | WF-B |
| I3 | 좌석 경합의 순위 척도는 후보 간 비교 가능해야 한다 | 실험 슬롯 간 tiebreak가 래깅 fit/align 포화 legacy conf (§1, E15) | WF-C |
| I4 | 백테스트가 검증 못한 성분은 라이브에서 점수를 못 움직인다 | whale conf에 c_fund 가산 잔존(설계 구간 분산 0, 라이브-온리, 티어 경계 이동 가능) | WF-D |
| I5 | 좌석 시간의 재배분은 전방 증거로만 | 실험 슬롯이 85% 좌석시간·2% 기여 (§2a) — 단 설계 구간 재적합은 금지 | Phase 4 |

### Phase 0 — 계측 (로직 불변, 즉시)

scoring.py 전방 리포트에 심사 지표 추가 — 이후 모든 승급/강등의 유일한 심판:
- 슬롯별 **좌석시간 점유율 vs R 기여율** (§2a의 전방 버전),
- **진입시점 hold_conf 분포**와 corr(hold_entry, R) (WF-A의 전방 검증),
- SignalFade/LossFade/BiasFlip **컷 직후 24/48h 가상 보유 R** (청산 다리별
  실효 — 아꼈는가 버렸는가),
- conf 분위-R 단조성 (E17 §3D 이월).

### Phase 1 — WF-A: 진입/홀드 정합 게이트 (측정 완료, §4 — 구현만 남음)

- 구현 지점: engine.py §3 후보→pending 단계. `hold_at[sym][d]`(결정봉)이
  `p.hold_conf_exit` 미만이면 후보 스킵. 무장 조건 `hold_conf_exit>0`이므로
  base/turbo/max는 자동 불변(비무장), whale만 발화.
- 기대: exp +0.196→+0.210R, ret 53→67.5x, 양쪽 반기↑ (§4 측정). floor
  0.40~0.55 전 구간 개선으로 강건성 기확인.
- 테스트: whale에서 hold<0.50 후보 미진입 / base 동일 후보 진입(비무장) /
  전 프로파일 회귀 수치.

### Phase 2 — WF-B: BiasFlip을 conviction 체계에 종속

- 진단(I2): BiasFlip은 hold_conf 도입 전의 이진 청산(E9)이다. HV-A 이후에도
  독립 발화해, 연속 판독이 "논지 생존"이라는 순간(44/45) 좌석을 내린다. 컷
  후 48h 순방향 +2.07% — 살아있는 추세를 이진 플리커로 하차.
- 변형 (제로-DOF): **hold 체계가 무장된 프로파일에서는**(hold_conf_exit>0)
  BiasFlip이 독립 청산이 아니라 conviction 붕괴의 **확인 신호**로 강등 —
  bias_flip_streak>=2 이면서 **동시에 hold_conf < hold_conf_exit**일 때만 발화.
  비무장 프로파일(base)은 현행 BiasFlip 유지(E9 채택 구조 불변).
- 주의(정직): BiasFlip 후 회전의 next_meanR은 +0.252로 나쁘지 않았다(§2d).
  이 변형의 손익은 "잘린 승자 계속 보유"와 "회전 기회 상실"의 차익 — 예단
  없이 반쪽 교차로 판정. 열위면 즉시 기각하고 현행 유지.
- 상호작용: WF-A 채택 후 스택으로 측정(진입이 깨끗해지면 BiasFlip의 방어
  가치가 변할 수 있음 — 순서 고정).

### Phase 3 — WF-C: 좌석 tiebreak 척도의 비교 가능화

- 진단(I3): whale fill 정렬 2차 키가 legacy conf. 검증/실험 간은 risk_scale
  1차 키가 해결(E15)했지만, **실험 슬롯끼리**(전부 0.20)는 래깅 라벨이
  포화시킨 conf가 좌석을 배정한다 — 서로 다른 자로 잰 점수의 경합.
- 변형 (제로-DOF): tiebreak를 `(risk_scale, 결정봉 hold_conf, conf)`로 —
  이미 모든 후보에 대해 계산되는 held-side conviction은 슬롯 무관 동일
  산식이라 비교 가능하고, "지금 좌석에 앉히면 유지 로직이 얼마나 신뢰하나"
  라는 좌석의 실제 질문과 일치한다.
- 판정: whale 반쪽 교차. WF-A가 바닥을 이미 자르므로 한계 기여가 작을 수
  있음 — 기여 0이면 단순성 원칙으로 기각(E15 V-C 전례).

### Phase 4 — I5: 좌석 시간 재배분 (전방 전용 — 설계 구간 재적합 금지)

- §2a의 좌석 왜곡(실험 85% 시간·2% 기여)의 근본 치료는 슬롯 강등이지만,
  실험 슬롯의 지위는 E12 프로토콜상 **전방 채점만이 심판**이다. 설계 구간
  수치로 PBK_L/RCL_S/BRK_S를 지금 끄는 것은 사후 선택(과적합)이라 하지
  않는다.
- 실행: Phase 0 계측이 슬롯별 좌석시간/R을 전방에서 8주 누적 → E12 기준
  (전방 8주 기대값 ≤0 → 강등/차단, n≥30 & >0 → 승급 심사)을 **좌석시간
  가중으로 확장**: 강등 기준에 "좌석 점유율 상위 & 기여 음수" 조건 추가
  여부를 8주 데이터가 모이는 시점에 별도 사전 등록.
- WF-A/B/C가 채택되면 실험 슬롯의 좌석 점유 자체가 줄어들므로(진입 바닥 +
  tiebreak), Phase 4의 부담은 그만큼 감소 — 순서가 곧 위험 관리다.

### WF-D — whale 펀딩 정합 (백테스트 불변, 라이브 전용 수정)

- 진단(I4): CV-A가 base에서 c_fund 가산을 제거·거부권화했지만 whale은
  legacy conf를 유지하므로 fm 가산이 남아 있다. 설계 구간에서 분산 0이라
  백테스트로는 영원히 검증 불가인 성분이, 라이브에서만 conf를 ±0.05/−0.10
  움직여 **레버리지 티어 경계(0.62/0.70/0.80)를 넘나들 수 있다** — 검증된
  티어 맵을 미검증 성분이 교란하는 유일한 경로.
- 수정 (측정 불가 → 불변성으로 판정): whale conf에서도 fm 가산 제거, CV-A와
  동일한 과밀 거부권 적용. **설계 구간 재현이 바이트 단위 불변임을 회귀로
  증명**(fm이 한 번도 발화 안 했으므로) — 즉 백테스트 위험 0, 라이브 동작만
  통일. 거부권의 실효는 Phase 0의 fund_z 발화 로그가 전방에서 판정.

### 실행 순서·중단 기준

```
0. 계측(Phase 0)            → 검증: 리포트 필드 생성, 로직 수치 불변
1. WF-A 구현               → 검증: §4 수치 재현 + 테스트 3종 + 전 프로파일 회귀
2. WF-D 구현               → 검증: whale 설계 구간 바이트 불변 + veto 테스트
3. WF-B 측정→채택/기각      → 검증: WF-A 스택 반쪽 교차, 열위 즉시 기각
4. WF-C 측정→채택/기각      → 검증: 기여 0이면 단순성 기각
5. Phase 4                 → 전방 8주 데이터 축적 후 별도 사전 등록
```

중단 기준: 어느 단계든 base 비무장 경로 수치가 변하면 그 자리에서 롤백.
whale MDD가 기준선(−46.5%) 대비 +10%p 이상 악화되는 채택은 없다(꼬리
프로파일이라도 진단서 §4의 비용 상한을 넘는 개선은 받지 않는다). 홀드아웃은
전 단계에서 미개봉 유지.
